# 开发历程与变更日志

本文件记录本仓库可核实的版本与开发里程碑。未创建 Git 提交或标签的工作统一标记为“未发布”，不追溯虚构版本号。

## 1.3.0 — 2026-07-13

### 新增

- 调色盘编辑窗口的背景主题会跟随当前预览额度实时变化，不再固定显示在 `50%`。
- 主悬浮卡片在焦点移至应用外部后自动收回 `100×100` 最小窗口。

### 变更

- 原生失焦事件增加短暂复核，焦点切换到色彩预览或调色盘编辑窗口时保持展开，避免应用内部操作误触发收起。
- 前端、Rust、Tauri 及锁文件版本统一升级至 `1.3.0`。

### 验证

- 前端测试、TypeScript/Vite 构建、Rust 检查及 Windows Tauri release 构建通过。
- 浏览器与真实 Windows Tauri 程序完成调色盘动态背景、应用内窗口切换保持展开及应用外失焦自动收起检查。

## 1.2.0 — 2026-07-13

### 新增

- 折叠悬浮窗与完整额度卡片改为同一 DOM 的 shared-layout morph：固定窗口锚点，连续过渡主数字、背景、圆角与阴影，并为卡片信息增加约 40ms 的错峰入场。
- 色彩预览与调色盘增加分阶段开关动画、五色块错峰、滑块圆点回弹及百分比平滑滚动。

### 变更

- 鼠标离开后延迟 120ms 收起；快速移入移出、快速开关调色窗口和动画中连续点击由可取消时序保护，避免闪烁与窗口残留。
- 调色窗口按预览先开、调色盘后开，关闭时反序且更快；退场结束后才隐藏原生窗口。
- Vite 开发服务忽略 Rust `target` 构建目录，避免 Windows DLL 链接期间触发 `EBUSY`。
- 所有新增动效支持 `prefers-reduced-motion`；前端、Rust、Tauri 及锁文件版本统一升级至 `1.2.0`。

### 验证

- 前端 17 项测试、TypeScript/Vite 构建、Rust 检查及 Windows Tauri release 构建通过。
- 浏览器完成 `100×100`、`320×320`、`320×104`、关键颜色、百分比滚动和减弱动画检查。
- 真实 Windows Tauri 程序完成三态切换、快速移入移出、连续开关、完成退出、双窗口跟随及屏幕底部上方避让验证。

## 1.1.0 — 2026-07-13

### 新增

- 会话 Token 浮窗支持直接拖动；拖动后保留相对 Codex 窗口的新位置，并继续随 Codex 窗口移动。

### 变更

- 移除 Token 浮窗的胶囊背景、边框、阴影和模糊效果，仅保留紧凑 Token 文字。
- 前端、Rust、Tauri 及锁文件版本统一升级至 `1.1.0`。

### 验证

- 前端 17 项测试、TypeScript/Vite 构建、Rust 检查及 Windows Tauri release 构建通过。
- 在真实 Windows Codex 窗口完成拖动保持和拖动后相对位置跟随验证。

## 1.0.0 — 2026-07-13

基线：上游 `v0.1.3`（`27943c0`）。以下内容构成 Quota Beacon `1.0.0` 的首个独立开发版本。

### 新增

- 本机 Codex 历史 Token 汇总与会话 Token 浮层显示。
- 101 级额度颜色主题：五个可编辑关键色、连续插值、对比度处理，以及防止相邻整数百分比重色的自动测试。
- 独立色彩预览窗口与调色盘编辑窗口；支持 `0–100%` 滑块预览、恢复默认色和保存自定义色。

### 变更

- 主卡片和折叠悬浮球改用同一套额度主题算法；以 5 小时额度为主视觉，缺少 5 小时数据时回退显示周额度。
- 真实额度改为每 10 秒强制刷新；预览期间同步显示滑块所选额度，退出后恢复真实数据。
- 增强双窗口协作、移动跟随、可用工作区上下避让、高 DPI 物理尺寸处理，以及预览期间的置顶状态恢复。
- 为额度数据、设置加载、Token 格式化和中英文文案补齐类型、桥接和测试覆盖。
- 进度条末端使用无实体的白色柔光提示当前位置，避免圆形标记造成“可拖动按钮”的错误暗示。
- 前端、Rust 与 Tauri 构建版本统一提升为 `0.1.4`，用于区分包含上述功能的可恢复版本。
